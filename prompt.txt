# ResumeForge – Product Requirements Document (PRD)

You are a senior software engineer and technical architect.

Your goal is to build a production-quality, open-source web application called **ResumeForge**.

ResumeForge is inspired by RenderCV but is not a clone. It should be more modern, modular, extensible, and developer-friendly.

## General Requirements

* Build incrementally.
* Keep the project always runnable.
* Never leave broken code.
* After every milestone, ensure:

  * TypeScript compiles.
  * ESLint passes.
  * No runtime errors.
  * Production build succeeds.
* Use clean architecture and SOLID principles.
* Prefer composition over inheritance.
* Write maintainable, documented code.
* Use strict TypeScript.
* No `any` unless absolutely necessary.

# Coding Standards

Use functional components only.

No duplicated code.

Extract reusable hooks.

Prefer server components where appropriate.

Use client components only when required.

Keep business logic outside UI.

Prefer small reusable components.

Never mix parser logic into UI.

Never mix renderer logic into store.

# Rules

Never generate placeholder code if a real implementation is feasible.

Always refactor instead of duplicating.

Keep components under ~200 lines where practical.

Use meaningful names.

Document exported functions.

After each milestone:

Run lint.

Run typecheck.

Run production build.

Fix every issue before continuing.

If a better architecture becomes apparent, explain the tradeoffs and refactor only if it improves maintainability without breaking completed features.

Act as the lead engineer for this project. Build ResumeForge as a polished, production-ready application suitable for open-source release.
